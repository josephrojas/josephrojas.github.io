¿Cómo funciona Docker?



Docker permite el manejo de entidades y comunicación desde REST API. Un contenedor puede ser mucho más liviano que una máquina vortual

Operaciones básicas

01

Operaciones como Correr, detener un contenedor, acceder a él, eliminar un contenedor, entre otros:




Inspeccionar un contenedor en ejecución

                    
                      docker inspect   <container_name>

Listar los contenedores

                    
                      docker inspect   <container_name>

Incluyendo el flag -a podemos listar los contenedores detenidos

Eliminar todos los contenedores detenidos

                    
                      docker prune

Ejecutar un contenedor

                    
                      docker run --name  <container_name>  <image_name>

Ejecutar un contenedor de fondo

                    
                      docker run -d --name  <container_name>  <image_name>

Ejecutar un comando dentro del contenedor

                    
                      docker exec -it <container_name> <command_to_execute>

Detener o iniciar un contenedor existente

                    
                      docker start|stop  <container_name>

Eliminar un contenedor

                    
                      docker rm <container_name>

Montar un contenedor con bind mount

                    
                      docker run --name  <container_name> -v <ruta_maquina:ruta_contenedor> <image_name>

Crear un volumen

                    
                      docker volume create <nombre_volumen>

Listar volúmenes

                    
                      docker volume ls

Montar un contenedor añadiendo un volumen

                    
                      docker run --name  <container_name> --mount src=<nombre_volumen>,dst=<ruta_contenedor> <image_name>

Eliminar un volumen

                    
                      docker volume rm <nombre_volumen>

Networking

03

Exponer contenedores

                      
                        docker run --name  <container_name> -p <puerto_maquina:puerto_contenedor> <image_name>

Listas interfaces de red

                      
                        docker network ls
    • bridge : retrocompatibilidad con versiones anteriores
      host: Permite comunicación con el host
      none: Deshabilita toda comunicación

    • Crear interfaz de red

                          
                        docker network create --attachable <nombre_red>




    Conectar un contenedor a una interfaz

                          
                        docker network connect <nombre_red> <container_name>

    Desconectar un contenedor a una interfaz

                          
                        docker network disconnect <nombre_red> <container_name>
